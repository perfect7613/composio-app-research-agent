Headline

Most apps are technically buildable. The hard part is credential access and trust boundaries.

Patterns

Where the work clusters

OAuth-heavy apps need consent design. API-key apps are faster to ship. Ads, fintech, and enterprise data platforms need admin review, partner approval, or stronger safety rails.

What I built

Seed

`data/apps.json` fixes the 100-app input set from the assignment.

Research

`npm run research` calls Firecrawl agent/search for official docs and draft structured rows.

Normalize

Rows are compressed into auth, gatedness, API surface, MCP status, verdict, blocker, and evidence.

Verify

`npm run verify:sample` can run Browser Use Cloud checks; the submitted sample shows hits and misses.

Ship

`npm run build` bundles data for this static slideshow. No backend is needed to review it.

Human judgment was used where docs were gated, product naming was ambiguous, or the agent over-generalized from adjacent APIs.

Verdict

Easy wins first, outreach second, unsafe writes last.

The best first wave is self-serve, broad API, low-risk read/write tooling: HubSpot, Pipedrive, Close, Intercom, Front, Slack, Discord, Mailchimp, Shopify, Firecrawl, GitHub, Airtable, Linear, Asana, Stripe, and similar apps. Ads, finance, enterprise data, and ambiguous AI-native apps need account access, app review, or partner outreach before engineering should spend heavily.
